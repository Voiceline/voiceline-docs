Credits are the usage-based component of VoiceLine pricing. They measure the value VoiceLine produces — not the time or effort that goes into producing it.

## How credits work

Every workspace has a shared credit pool. Credits are drawn from that pool across three categories: workflow outputs, live call minutes, and analytics processing. All users draw from the same pool.

**Workflow outputs** — when VoiceLine produces a CRM record or follow-up action (visit report, opportunity, task, and so on), credits are charged per output at a fixed weight. Higher-value outputs like opportunity creation cost more credits than utility actions like setting a reminder.

**Live call minutes** — when reps use the phone-based assistant, each minute of the live call consumes credits in addition to any workflow outputs the call produces.

**Analytics processing** — insight extraction from recordings consumes credits per extracted insight. AI Analyst query credits will apply in a future release.

See [Credit Consumption](/commercial/credits/consumption) for the full weight schedule across all three categories.

## Credit baseline

FSF includes **500 credits per active user per month**, pooled at the workspace level. The baseline is recalculated monthly based on active user count and resets each month — unused baseline credits do not carry over.

| Team size | Monthly baseline |
|-----------|-----------------|
| 15 users | 7,500 credits |
| 30 users | 15,000 credits |
| 100 users | 50,000 credits |
| 300 users | 150,000 credits |

If your team's usage exceeds the monthly baseline, you can purchase additional credit packages. Packages are also calculated monthly — unused credits from a package do not roll over, so it's worth estimating your team's typical output volume before selecting a package size.
