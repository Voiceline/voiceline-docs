# _internal/

Working files used by maintainers and AI agents. **Not published** to the documentation site.

| File | Purpose |
|------|---------|
| `feature_map.md` | Master index of all 45+ pages — tracks research and writing status per page |
| `blank_spots.md` | Known gaps, open questions, and missing sources — routed by audience (CPO / Dev / Designer) |
| `cross_ref_queue.md` | Incidental findings about features not in the current batch — forward-carry to next session |
| `sources_scanned.md` | Log of every source file/Figma/Notion page read — prevents re-reading the same source |
| `docs_project.md` | Full project history — phases, batch log, decision log, open questions |
| `hierarchy_draft.md` | CPO-provided feature hierarchy — authoritative source for product area structure |
| `figma_links.md` | Figma URLs per feature — used during documentation research |

## For AI agents

Start with `feature_map.md` to understand what exists and what's missing before making changes.
Check `blank_spots.md` before asking questions the CPO may have already answered.
Log new gaps to `blank_spots.md` rather than leaving TODOs inline in content files.
