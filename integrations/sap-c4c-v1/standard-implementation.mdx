This page documents the standard VoiceLine FSF implementation on SAP C4C v1 — the default configuration your SA will deploy, and the decisions that need to be made during scoping.

## SAP-specific implementation notes

SAP C4C v1 has a specific object model and naming convention that differs from other CRMs. The key mappings for field sales teams are:

| VoiceLine workflow | SAP object | SAP type |
|-------------------|------------|----------|
| Visit Report | Appointment | Configurable type (e.g. "Customer Visit") |
| Call Log | Phone Call Activity | Configurable type |
| Task | Task | Standard |
| Meeting | Appointment | Configurable type (e.g. "Customer Meeting") |
| Opportunity | Opportunity | Standard |
| Lead | Lead | Standard |
| Contact | Individual (Contact) | Standard |

## Required scoping decisions

Before implementation begins, your SA needs alignment on the following. These are SAP-specific decisions that cannot be defaulted.

### 1. Appointment type values
SAP C4C uses configurable "appointment types" to classify activities. VoiceLine's Visit Report and Meeting workflows map to SAP Appointment objects — your SA needs to know which appointment type values to use.

**Decision required:** What appointment type should Visit Reports map to? What type should Meetings map to?

### 2. Service user permissions
The SAP service user must have read/write permissions for all objects VoiceLine syncs. IT must confirm the service user's permission set before connection testing.

**Decision required:** Service user created and permissions confirmed by IT.

### 3. Account scope per rep
SAP C4C assigns accounts to reps via the OwnerID field. VoiceLine uses this to filter which accounts each rep sees. If your SAP assigns accounts differently (e.g. by territory group, not individual owner), the lookup filter needs to be configured accordingly.

**Decision required:** How are accounts assigned to reps in SAP? Owner assignment, territory group, or other?

### 4. Network access
On-premise or private-network SAP deployments require VPN access or IP whitelisting for VoiceLine's servers.

**Decision required:** Is your SAP accessible from the internet, or does it require VPN/IP whitelist? If VPN is required, this is a non-standard implementation (billed at €1,300/day).

## Go-live checklist

- [ ] Service user created in SAP with correct API permissions
- [ ] SAP tenant URL confirmed and accessible
- [ ] Appointment type values agreed and mapped
- [ ] Account scope logic confirmed
- [ ] Connection test passed in VoiceLine workspace settings
- [ ] Initial sync completed (accounts and contacts visible in app)
- [ ] Test visit report submitted and verified in SAP
- [ ] Test call log submitted and verified in SAP
- [ ] Test task submitted and verified in SAP
- [ ] Pilot rep group onboarded and using workflows

## Common SAP C4C v1 implementation issues

**Appointment type not found:** If the appointment type value in VoiceLine does not match an active type in SAP, records will fail to create. Confirm exact type values with your SAP administrator.

**Account lookup returning no results:** Verify that the service user has read access to Account objects and that accounts are assigned to the rep's user ID in SAP.

**VPN connection failures:** If the connection test fails with a network error, the issue is almost always network access. Work with your IT team to whitelist VoiceLine's IP range.
