This page documents what data flows between VoiceLine and SAP C4C v1, in which direction, and on what schedule.

## Sync overview

| Object | Direction | Trigger |
|--------|-----------|---------|
| Accounts | SAP → VoiceLine | On login + scheduled refresh |
| Contacts | SAP → VoiceLine | On login + scheduled refresh |
| Leads | SAP → VoiceLine | On login + scheduled refresh |
| Opportunities | SAP → VoiceLine | On login + scheduled refresh |
| CRM users (employees) | SAP → VoiceLine | On login |
| Visit reports | VoiceLine → SAP Appointment | On submission |
| Call logs | VoiceLine → SAP Phone Call Activity | On submission |
| Tasks | VoiceLine → SAP Task | On submission |
| New leads | VoiceLine → SAP Lead | On submission |
| New contacts | VoiceLine → SAP Contact | On submission |

## Sync cadence

| Direction | Trigger | Details |
|-----------|---------|---------|
| SAP → VoiceLine | On rep login | Full refresh of assigned accounts, contacts, leads, opportunities |
| SAP → VoiceLine | Scheduled | Background refresh — interval configured during implementation (typically 4–6 hours) |
| VoiceLine → SAP | On submission | Immediate; queued and retried if SAP is unreachable |

## What VoiceLine does not sync

- Financial records (orders, invoices, contracts)
- SAP custom objects not in the standard data model
- Email content
- SAP workflow approvals or status triggers

<Note>Field-level mappings shown here are the standard defaults. Your SA may configure different mappings during implementation to match your SAP field names and picklist values. Deviations from the standard are documented during your implementation scoping.</Note>
