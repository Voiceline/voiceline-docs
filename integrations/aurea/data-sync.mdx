This page documents what data flows between VoiceLine and Aurea CRM, in which direction, and on what schedule.

## Sync overview

| Object | Direction | Trigger |
|--------|-----------|---------|
| Companies (Accounts) | Aurea → VoiceLine | On login + delta refresh |
| Persons (Contacts) | Aurea → VoiceLine | On login + delta refresh |
| Visit reports | VoiceLine → Aurea Appointment | On submission |
| Call logs | VoiceLine → Aurea Call | On submission |
| Tasks | VoiceLine → Aurea Task | On submission |

## Aurea data model note

Aurea uses "Company" for accounts and "Person" for contacts. VoiceLine maps these internally — reps see "Account" and "Contact" in the app.

Aurea uses a query-based REST API with delta sync support. VoiceLine fetches only records updated since the last sync, using Aurea's `Upd` and `UpdTime` fields as markers.

## Sync cadence

| Direction | Trigger | Details |
|-----------|---------|---------|
| Aurea → VoiceLine | On rep login | Delta sync — only records updated since last sync |
| Aurea → VoiceLine | Scheduled | Delta refresh — interval configured during implementation |
| VoiceLine → Aurea | On submission | Immediate; queued and retried if Aurea is unreachable |

## What VoiceLine does not sync

- Aurea financial records or order data
- Custom Aurea modules outside the standard Company/Person/Activity model

<Note>Aurea implementations are workspace-specific. Field configurations vary by customer deployment. Your SA will confirm the exact field mapping for your Aurea instance during scoping.</Note>
