This page documents what data flows between VoiceLine and ZOHO CRM, in which direction, and on what schedule.

## Sync overview

| Object | Direction | Trigger |
|--------|-----------|---------|
| Accounts | ZOHO → VoiceLine | On login + scheduled refresh |
| Contacts | ZOHO → VoiceLine | On login + scheduled refresh |
| Deals | ZOHO → VoiceLine | On login + scheduled refresh |
| CRM owners | ZOHO → VoiceLine | On login |
| Visit reports | VoiceLine → ZOHO Event (Appointment) | On submission |
| Call logs | VoiceLine → ZOHO Call | On submission |
| Tasks | VoiceLine → ZOHO Task | On submission |
| Notes | VoiceLine → ZOHO Note on Account/Contact | On submission |

## Sync cadence

| Direction | Trigger | Details |
|-----------|---------|---------|
| ZOHO → VoiceLine | On rep login | Full refresh of accounts, contacts, deals |
| ZOHO → VoiceLine | Scheduled | Background refresh — interval configured during implementation (typically 4–6 hours) |
| VoiceLine → ZOHO | On submission | Immediate |

## What VoiceLine does not sync

- ZOHO Desk tickets
- ZOHO Campaigns or Marketing Hub data
- Custom ZOHO modules (unless configured as a non-standard implementation)

<Note>Field-level mappings shown here are the standard defaults. Your SA may configure different mappings during implementation to match your ZOHO field names and pipeline stage values.</Note>
