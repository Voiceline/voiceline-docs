Configure the languages reps work in across the VoiceLine workspace.

## Working language

Each rep is configured with one or more working languages — the language they speak into VoiceLine, see the interface in, and receive AI-generated summaries and CRM field values in. Input and output are always in the same language: a rep who records in French gets French summaries and French CRM entries.

Reps who work across multiple countries can be assigned multiple working languages and switch between them as needed. Your workspace admin activates the available languages during setup; each rep then selects their own language during onboarding.

## CRM translation

<Note>CRM translation is an Enterprise feature. Contact your account manager to enable it.</Note>

By default, CRM records are written in the rep's working language. CRM translation changes this: when a rep submits a recording, their AI-generated output is written in their working language as usual — but when the record syncs to the CRM, it is automatically translated into the workspace's configured CRM language.

This is useful when a central team or manager needs all CRM records in a single language, but reps across countries record in their own languages.

## Custom vocabulary

Workspace-specific terms the assistant should recognise:

**Product aliases** — alternative names for products in your catalogue. If a product is officially called "Model X-2000" but reps refer to it as "the 2000", add the alias so search resolves correctly.

**Custom terms** — technical terms, internal acronyms, or brand names. Adding these to the vocabulary reduces transcription errors for specialist terminology.

Go to **Workspace Settings → Product Recognition → Custom Terms** or **Product Aliases** to manage these.

## FAQ

**Can a rep record in a different language than other reps in the same workspace?**
Yes — working languages are configured per user. Different reps or user groups can have different working languages within the same workspace.

**Does changing the working language affect historical recordings?**
No — language changes apply to new recordings only. Historical reports remain in the language they were originally generated in.
