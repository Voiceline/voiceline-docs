Connects VoiceLine to SAP C4C using the v2 API, supporting account, contact, and opportunity sync alongside outbound field activity records.

## Overview

VoiceLine pulls accounts, contacts, and opportunities from SAP C4C v2 so reps can search and reference their CRM data in the field. When reps submit field activities — visit reports, call logs, tasks, or new accounts and contacts — VoiceLine pushes these records back to SAP as structured CRM objects.

## Where to configure it

Go to **Workspace Settings** → **Integrations** → **SAP Cloud for Customer (v2)**.

## Setup

**Prerequisites:**
- SAP C4C v2-compatible tenant URL
- Service user credentials (username + password) with v2 API access
- Network access to your SAP tenant (VPN may be required — see [Integrations](/customer/platform/integrations/index#network-requirements))

**Steps:**
1. Open **Workspace Settings** → **Integrations**
2. Select **SAP Cloud for Customer (v2)**
3. Enter your SAP tenant URL, username, and password
4. Click **Test connection**
5. Save — VoiceLine begins the initial sync

## Sync behavior

| Object | Direction | When |
|--------|-----------|------|
| Accounts | SAP → VoiceLine | On login / scheduled refresh |
| Contacts | SAP → VoiceLine | On login / scheduled refresh |
| Opportunities | SAP → VoiceLine | On login / scheduled refresh |
| Visit reports | VoiceLine → SAP (Appointment) | On submission |
| Call logs | VoiceLine → SAP (Call Log / Note) | On submission |
| Tasks | VoiceLine → SAP (Task) | On submission |
| New accounts | VoiceLine → SAP | On submission |
| New contacts | VoiceLine → SAP | On submission |

## Authentication

SAP C4C v2 uses **Basic authentication** — service user credentials stored in VoiceLine's integration settings, encrypted at rest.

<Note>A dedicated SAP service user is required. The service user must have API permissions for the objects listed above.</Note>

## FAQ

**What is the difference between SAP C4C v1 and v2?**
v1 uses the older OData API surface. v2 uses a newer API with different endpoints and object models. The right version depends on your SAP C4C deployment and API configuration — consult your SAP administrator.

**Can new accounts and contacts created in VoiceLine be pushed to SAP?**
Yes — when a rep creates a new account or contact via VoiceLine, the record is pushed to SAP C4C on submission (subject to deduplication checks — see [Customer Data Quality](/customer/productivity/customer-data-quality/index)).
