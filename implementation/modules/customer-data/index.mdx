The Customer Data module adds structured creation and update workflows for contacts, accounts, and leads. Reps capture new contact details, update account information, and qualify leads by voice — with deduplication logic that prevents duplicate records from being created in your CRM.

For a full description of what the module includes commercially, see [Customer Data](/commercial/modules/customer-data).

## What it activates

| Capability | Description |
|-----------|-------------|
| Contact creation | Create a new contact record in the CRM by voice |
| Account creation | Create or update an account record by voice |
| Lead creation | Create and qualify a new lead by voice |
| Deduplication | Duplicate detection and resolution before CRM write |

The Contact, Account, and Lead workflows are available in FSF. Customer Data extends them with deduplication logic and the Customer Data Quality features in the analytics surface.

## Contact deduplication

Before creating a new contact, VoiceLine checks for existing records using the configured match criteria. Default match criteria:

| Match criteria | Default behaviour |
|----------------|-------------------|
| First name + last name + account | Exact or fuzzy match flags as potential duplicate |

Thresholds are configurable. Confirm the threshold with your CRM admin based on the quality of your existing contact data — stricter thresholds reduce false negatives but may surface more false positives for reps to resolve manually.

## Required scoping decisions

**Deduplication threshold:** Confirm the match threshold based on your CRM's contact data quality. Define what happens when a duplicate is found (prompt for confirmation, or block creation).

**Mandatory fields:** Agree which fields are required before a record can be submitted. Requiring email address reduces duplicates but may block rep workflow if email is unknown.

**Lead qualification criteria:** If your CRM has a lead qualification stage model, confirm how the Lead workflow maps to your qualification fields.

## Dependencies

Customer Data has no module dependencies. It can be activated as the first module after FSF.
