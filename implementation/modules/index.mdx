Add-on modules extend VoiceLine beyond Field Sales Fundamentals. Each module activates additional workflows, data capture capabilities, or analytics surfaces — and each one has a defined standard implementation that your SA deploys.

Modules are activated after your FSF rollout is live. See [Phase 2 — Module Expansion](/implementation/our-approach/phase-2-expansion) for how module rollouts work.

## Available modules

| Module | What it adds |
|--------|-------------|
| [Pipeline Management](/implementation/modules/pipeline-management) | Opportunity capture and pipeline tracking by voice |
| [Customer Data](/implementation/modules/customer-data) | Contact, account, and lead creation with deduplication |
| [Marketing Surveys](/implementation/modules/marketing-surveys) | Structured voice-based survey completion |
| [Service Cases](/implementation/modules/service-cases) | Field case creation and service technician workflows |
| [Market Analytics Pro](/implementation/modules/market-analytics-pro) | Custom competitor, product, and industry signal tracking |
| [Sales Coaching Pro](/implementation/modules/sales-coaching-pro) | Playbooks, coaching dashboards, and rep performance tracking |
| [Industry Fairs](/implementation/modules/industry-fairs) | Lead capture and visit workflows at trade shows and events |
| [Inside Sales](/implementation/modules/inside-sales) | Call-centre and inside sales team workflows |

## How module implementations work

Each module page documents:
- **What the module activates** — which new workflows or capabilities become available
- **Standard implementation** — the default configuration your SA deploys
- **Required scoping decisions** — what needs to be agreed before configuration begins
- **Dependencies** — other modules or configurations that must be in place first

For commercial details on each module, see the [Modules & Pricing](/commercial/modules) tab.
