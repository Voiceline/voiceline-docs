VoiceLine reads open tasks from the CRM and writes new tasks back on submission. For how tasks work in VoiceLine, see [Tasks](/customer/platform/data-objects/tasks).

By the end of this page your FDE will have confirmed with your CRM admin:

- Which task type values represent field sales tasks (to scope the briefing correctly)
- The sync filters to apply (by type, assigned rep, linked account)
- Whether task auto-creation from visit reports is needed

## Fields synced from CRM

| Field | Required | Notes |
|-------|----------|-------|
| Title | Required | Displayed as the task heading in the briefing card |
| Task description | Required | Displayed in the briefing task card |
| Due date | Required | Briefing task card, overdue indicator |
| Assigned rep | Required | Rep-scoped task visibility |
| Linked account | Required | Account-scoped task lookup |
| Completion status | Required | Determines whether task appears in briefing |
| Priority | Optional | Displayed in the briefing task card |
| Linked contact | Optional | Contact display in task card |
| Linked call log | Optional | Associates the task with a call record |
| Linked visit | Optional | Associates the task with a visit record |

## History window

The default history window is **12 months**. A longer or shorter window can be configured — discuss with your FDE. A very large history window increases sync volume and may affect performance.

## Sync filters

| Filter | Description |
|--------|-------------|
| Open tasks only | Only sync tasks that are not yet completed — recommended |
| Assigned to rep | Only surface tasks assigned to the viewing rep (not team-level tasks) |
| Linked to account | Only surface tasks linked to an account in scope |
| By task type | Filter to specific task types relevant to field sales (e.g. exclude IT or internal tasks) |
