Industries in VoiceLine are used for segmentation in analytics — specifically in Industry Analytics and as a filter dimension in Visit Analytics and coaching dashboards. Reps do not interact with industries directly; the industry value on an account is synced from the CRM and used automatically.

For the customer-facing description of industries in VoiceLine, see [Industry Analytics](/customer/analytics/industry-analytics).

## How industries work

Each account in VoiceLine can have one or more industry values. Industries are one-dimensional — VoiceLine does not support hierarchical industry trees (e.g. "Manufacturing > Automotive > Tier 1 Supplier"). Only flat values are supported.

Multiple industries per account are supported. An account can be tagged with several industry values if it operates across sectors.

## How industry data is sourced

There is no standard CRM integration for industries — setup is custom for every customer. The FDE configures which CRM field to read and which values to map to VoiceLine's industry taxonomy.

The process:

1. **Identify the CRM field** — Confirm which field on the account object holds the industry value (e.g. `Industry`, `Sector`, `AccountCategory`)
2. **Agree the taxonomy** — Define the list of industry values VoiceLine will use. These can match your CRM values exactly, or be a normalised set the CRM values are mapped to
3. **Configure the mapping** — If your CRM uses internal codes or values that differ from the names you want in analytics (e.g. CRM stores "MFG" → should display as "Manufacturing"), your FDE configures the mapping

## Normalisation considerations

If different CRM regions or business units use different industry terminology for the same sector, normalisation is required before meaningful analytics are possible. Your FDE will surface this during the scoping conversation.

Examples where normalisation is needed:
- German entity uses "Lebensmittel", English entity uses "Food & Beverage" — both should map to one industry in VoiceLine
- CRM stores numeric codes (e.g. NACE codes) — these need mapping to display names

## Scoping decisions

Before your FDE configures industries, confirm the following:

- Which CRM field(s) hold industry values on the account object
- Whether the CRM values should be used as-is or normalised to a standard taxonomy
- The final list of industry values to use in VoiceLine analytics
- How to handle accounts with no industry value in the CRM (exclude from industry analytics, or assign a default "Unknown" value)
