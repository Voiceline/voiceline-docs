Admin and security configuration determines how users authenticate to VoiceLine, how the platform connects to your CRM, and what security and access controls apply to your workspace.

## Authentication

VoiceLine supports two authentication methods:

| Method | Description | When to use |
|--------|-------------|-------------|
| Basic auth | Email and password, managed in VoiceLine | Standard for most deployments; no IdP dependency |
| SSO (SAML 2.0) | Single sign-on via your identity provider | Enterprise customers with an existing IdP (Azure AD, Okta, etc.) |

SSO requires your IT team to configure a SAML application in your identity provider and provide the metadata URL or XML to your SA. This is confirmed during scoping and configured before go-live.

## CRM connection

VoiceLine connects to your CRM via OAuth. During setup, a service account or CRM administrator account is used to authorise the connection. The OAuth token is stored securely and used for all data sync operations.

**Required CRM permissions:**
- Read access to accounts, contacts, leads, opportunities, products, competitors, and users
- Write access to the activity objects that VoiceLine creates (visit reports, call logs, tasks, etc.)
- Write access to any CRM objects created by active modules (leads, contacts, opportunities)

Your SA will provide the exact permission set required for your CRM.

## VPN and network requirements

VoiceLine is a cloud-based service. By default, it does not require VPN access. If your CRM is hosted on-premise or behind a corporate network, the following must be confirmed:

- Whether VoiceLine can reach the CRM endpoint from the internet
- Whether a VPN or IP allowlist is required for the CRM API
- Whether the mobile app requires VPN to connect to VoiceLine services

<Note>On-premise CRM deployments (e.g. Aurea CRM) require network access from VoiceLine's cloud infrastructure. Confirm with your IT team before configuration begins.</Note>

## Mobile device management (MDM)

If your organisation uses MDM to manage employee devices, confirm the following before go-live:

- The VoiceLine mobile app is available on both App Store and Google Play — confirm your MDM policy allows installation
- If app distribution is managed via MDM, your SA can provide the bundle ID and metadata required for managed deployment
- Push notification permissions are required for follow-up reminders and task notifications

## Attachment handling

VoiceLine supports attaching files to visit reports and other workflow submissions. The following must be confirmed:

| Setting | Options |
|---------|---------|
| Attachment types allowed | All file types / images only / none |
| Storage location | VoiceLine-hosted (default) / CRM-linked |
| Max file size | Configurable — default is 25 MB |

## Sync cadence

VoiceLine syncs CRM data on a configurable schedule. The standard cadences are:

| Object type | Default sync cadence |
|-------------|---------------------|
| Accounts | Every 4 hours |
| Contacts | Every 4 hours |
| Leads | Every 4 hours |
| Opportunities | Every 4 hours |
| Products | Daily |
| Competitors | Daily |
| Users | Daily |

Sync cadence can be adjusted per workspace. Near-real-time sync (15–30 minute intervals) is available for accounts and contacts where rep-facing data freshness is critical.
