The Reminder workflow lets reps set personal reminders by voice — lightweight personal action items that do not necessarily require a CRM record.

## What it captures

| Field | Type | Description |
|-------|------|-------------|
| Reminder text | Text | What the rep needs to remember to do |
| Due date / time | DateTime | When the reminder should fire |
| Linked account | Lookup (optional) | Account context for the reminder |
| Priority | Picklist | e.g. high, normal |

## Notification behaviour

Reminders fire as push notifications in the VoiceLine mobile app at the specified time. They do not generate calendar events unless explicitly linked.

## CRM sync

Reminders are personal by default and do not sync to the CRM. If CRM task creation is required, use the [Task workflow](/implementation/standard-implementations/task) instead.

Optional CRM sync (creating a Task in the CRM from a Reminder) is available as a configuration option — disabled by default.

## How your SA tailors this

- **CRM sync** — enable optional Task creation in CRM on reminder creation
- **Default reminder offset** — set a default lead time (e.g. 1 day before) for reminders created without an explicit time
- **Account linkage** — make account linkage required for reminders above a certain priority

## FAQ

**What is the difference between a Reminder and a Task?**
A Reminder is personal — it fires as a push notification in VoiceLine and is not (by default) visible to anyone else or tracked in the CRM. A Task creates a record in the CRM that is trackable by managers and syncs across systems. Use Reminders for quick personal prompts; use Tasks for anything that needs to be tracked or delegated.

**Can a Reminder also create a CRM task?**
Yes — optional CRM task creation is a configurable setting. Your SA can enable this during setup if your team prefers all action items to appear in the CRM.
