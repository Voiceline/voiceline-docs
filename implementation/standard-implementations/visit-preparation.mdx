Visit Preparation (Visit Briefing) gives reps a pre-visit summary before they start a recording. It surfaces recent activity, open tasks, and contact information for the account they are about to visit.

For a full description of what Visit Preparation shows to the rep, see [Visit Briefing](/customer/productivity/visit-briefing).

## What this means for implementation

Visit Preparation requires no configuration as part of a standard FSF implementation. It is automatically available to all reps once the workspace is set up and CRM data objects are synced.

The information displayed in Visit Preparation is drawn entirely from synced CRM data and prior VoiceLine activity. No workflow schema configuration is required.

## Extending Visit Preparation

The standard Visit Preparation surfaces the default data set. If you want to display additional context — for example, open orders from an ERP, contract data from a different system, or custom account fields not in the standard sync — this requires FDE involvement.

<Note>If you want to show additional data sources or custom context in Visit Preparation, contact your FDE to discuss what is possible and what is in scope for your implementation.</Note>

Visit Preparation has no workflow schema and therefore no component library configuration. For how the underlying data is configured, see the [CRM Configuration](/implementation/standard-implementations/crm-accounts) pages.
