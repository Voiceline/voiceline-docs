<Update label="2026-04-22" tags={["Improvement"]} rss={{ title: "Localized CRM error messages" }}>
  ## Localized CRM error messages

  When a CRM sync fails, error messages are now displayed in your configured language instead of raw technical output. This applies to all supported CRM systems — Salesforce, Microsoft Dynamics, HubSpot, SAP, Aurea, and ZOHO.

  **What changed:**
  - CRM sync errors now appear in your language — 13 languages are supported, including English, German, French, Spanish, Italian, Dutch, Czech, Croatian, Hungarian, Turkish, Polish, Romanian, and Portuguese
  - Common errors show specific, actionable guidance: permission issues, outdated meetings, missing CRM objects, VPN connectivity problems, future-dated meetings, and field value length limits
  - Unrecognized CRM errors are wrapped in a clear format that explains what happened and how to proceed, rather than showing raw system output

  No action is required — these improvements apply automatically to all workspaces with a CRM integration.
</Update>

<Update label="2026-04-21" tags={["Improvement"]} rss={{ title: "Improved event selection accuracy and language support" }}>
  ## Improved event selection accuracy and language support

  Event selection — the step where you link a visit report or workflow to a CRM calendar event — now produces more accurate AI-generated fields. Titles, time ranges, and other event details are generated correctly across all supported languages.

  **What changed:**
  - AI-generated event titles and time ranges are now more accurate when creating or matching CRM events
  - Event fields are generated in your configured working language, fixing cases where multilingual workspaces saw incorrect or mixed-language output
  - Pre-planned visits with a linked calendar event are matched more reliably

  No action is required — these improvements apply automatically to all workspaces using event selection.
</Update>
